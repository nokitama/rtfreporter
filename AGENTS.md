# Agent Instructions for rtfreporter

## ⚠️ Mandatory first step

**Before modifying any code, test, or documentation, always read the spec files first.**

```
vignettes/articles/internal-design.qmd   ← Internal R6 class design
vignettes/articles/external-api.qmd      ← Public S3 API contracts
specs/release_guidelines.md              ← Release and packaging rules
```

The spec documents are the **single source of truth**. All R code, tests, and
vignettes must conform to the specs. If a spec and code disagree, the spec wins
unless the user explicitly asks to change the spec first.

---

## Workflow rule

1. **Read relevant spec sections** before writing any code.
2. **Update specs first** when a design decision changes (API shape, behavior,
   defaults, data structures).
3. **Then update** in this order:
   - R implementation (`r/`)
   - R help (`man/`, roxygen comments in `r/`)
   - R vignettes (`vignettes/`)
   - R tests (`tests/`)
4. Never introduce an API pattern in code that is not documented in the specs.

## Key API rules (summary — see specs for full detail)

### Public API: S3 pipe pattern

The primary user-facing API uses the pipe (`%>%`) pattern:

```r
report <- rtf_document() %>%
  rtf_section(page = 1, secinfo = list(header = hdr, footer = ftr)) %>%
  rtf_tables(list(rtftable(df1), rtftable(df2))) %>%
  generate_rtfreport("output.rtf", overwrite = TRUE)
```

### Internal class naming

Internal R6 classes use the `_r6` suffix:
- `rtfreport_r6` — document object
- `rtftable_r6` — table content
- `rtfplot_r6` — figure content

Never reference these in exported functions or documentation.

### Border class hierarchy

```r
# Single edge
s <- rtf_border_side(style = "single", width = 15L, color = NULL)

# Four edges
b <- rtf_border(top = s, bottom = NULL, left = NULL, right = NULL)

# Convenience
rtf_border_none()
rtf_border_top(style, width, color)
rtf_border_bottom(style, width, color)
rtf_border_box(style, width, color)

# Per-zone table border
tb <- rtf_table_border(header=..., spanning=..., body=..., first_row=..., last_row=...)
rtf_border_tfl()   # Clinical TFL preset
```

### Header/Footer constructors

```r
hdr <- rtf_header(
  rows = list(
    c(l = "Protocol: RTF-101", r = "Company"),
    c(l = "Table 14.1.1",     r = "Page {AUTO_PAGE} of {AUTO_TOTAL_PAGES}")
  )
)
ftr <- rtf_footer(c(l = "Confidential"))
```

### Page tokens

| Token | Behavior |
|-------|----------|
| `{AUTO_PAGE}` | `\chpgn` — dynamic page number (recommended) |
| `{AUTO_TOTAL_PAGES}` | RTF NUMPAGES field — dynamic total (recommended) |
| `{PAGE}` | Alias for `{AUTO_PAGE}` |
| `{TOTAL_PAGES}` | Static total page count at render time |

Recommended: `"Page {AUTO_PAGE} of {AUTO_TOTAL_PAGES}"`.

## Vignette rendering (pre-built HTML)

```powershell
cd C:\Yrepo\rtfreporter

# 1. Build vignettes
Rscript -e "devtools::build_vignettes()"

# 2. Copy to inst/doc/
Copy-Item "doc\*" "inst\doc\" -Recurse -Force

# 3. Verify
Rscript -e "devtools::install(); vignette(package='rtfreporter')"
```

Articles (in `vignettes/articles/`) are Quarto documents rendered separately:

```powershell
cd C:\Yrepo\rtfreporter\vignettes\articles
quarto render internal-design.qmd
quarto render external-api.qmd
```
